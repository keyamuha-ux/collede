Introduction

Collede is a high-performance AI Gateway designed to provide a unified interface for multiple LLM providers. It is fully compatible with the OpenAI API specification, allowing you to switch providers by simply changing your base URL.

The gateway acts as a secure proxy. Your API keys are stored safely on the Collede server and are never exposed to your client-side applications.

Authentication

Collede uses secure session management. When making requests to the gateway, you must include your session token in the Authorization header. 

Authorization: Bearer YOUR_SESSION_TOKEN

Base URL

All API requests should be directed to the following base URL: 

https://your-collede-instance.com/v1

Chat Completions

Create a model response for the given chat conversation. This endpoint supports both standard JSON responses and Server-Sent Events (SSE) for streaming.

POST /v1/chat/completions

Python SDK Example 

from openai import OpenAI

client = OpenAI(
    base_url="https://your-collede-instance.com/v1",
    api_key="YOUR_SESSION_TOKEN" # Use your session token here
)

response = client.chat.completions.create(
    model="gpt-4",
    messages=[{"role": "user", "content": "Hello!"}]
)

print(response.choices[0].message.content)
